# ĐẶC TẢ DỮ LIỆU / DATA DICTIONARY
## File: sme_loan_customers_30000_samples.csv

## 1) Tổng quan / Overview

### Tiếng Việt
Bộ dữ liệu được sinh theo nguyên tắc nhân quả:
Feature (hành vi kinh doanh) -> Component scores -> CIC_SCORE -> Label.

Điều này có nghĩa là các đặc trưng được tạo trước, điểm tín dụng được tính sau, và nhãn rủi ro được gán cuối cùng; không tạo feature theo nhãn (không target-driven bias).

Riêng file 30000 mẫu này đã bật cơ chế quantile quota trong notebook, nên:
- label: nhãn sau khi cân bằng theo quota quantile.
- label_cic_range: nhãn gốc theo dải điểm CIC trước khi ghi đè.
- label_quantile: nhãn quantile trung gian dùng để phân bổ quota.

### English
This dataset is generated using a causal pipeline:
Feature (business behavior) -> Component scores -> CIC_SCORE -> Label.

This means features are generated first, credit score is computed next, and risk label is assigned last; no feature is generated from target labels (no target-driven bias).

For this 30,000-row file, quantile quota was enabled in the notebook, so:
- label: final quota-balanced label.
- label_cic_range: original label based on CIC score range before overwrite.
- label_quantile: intermediate quantile label used for quota allocation.

## 17. CV_score

- Bản chất: điểm ổn định dòng tiền; CV càng cao thì điểm càng thấp.
### EN
- Source: _calculate_financial_metrics().

- Extra penalty: if CV_value > 0.6, subtract 0.10 (floored at 0).

## 23. CIC_SCORE
### VI
- Nguồn: _calculate_cic_score().
- Công thức trọng số:
  cic_raw =
    0.15 * Growth_score +
    0.30 * CV_score +
    0.10 * Spike_score +
    0.25 * Txn_freq_score +
    0.10 * Years_score +
    0.10 * Industry_score
- Sau đó:
  - clip cic_raw vào [0,1]
  - cộng noise Gaussian N(0, 0.03), clip lại [0,1]
  - đổi thang: CIC_SCORE = 150 + 600 * cic_raw
  - làm tròn 2 chữ số thập phân.
- Bản chất: điểm tín dụng tổng hợp toàn cục theo thang 150-750.
### EN
- Source: _calculate_cic_score().
- Weighted formula:
  cic_raw =
    0.15 * Growth_score +
    0.30 * CV_score +
    0.10 * Spike_score +
    0.25 * Txn_freq_score +
    0.10 * Years_score +
    0.10 * Industry_score
- Then:
  - clip cic_raw to [0,1]
  - add Gaussian noise N(0, 0.03), clip again
  - scale: CIC_SCORE = 150 + 600 * cic_raw
  - round to 2 decimals.
- Nature: final global credit score in 150-750 scale.

## 5) Khuyến nghị sử dụng / Usage Recommendations

### VI
- Nếu bạn cần nhãn phản ánh đúng dải CIC tuyệt đối, ưu tiên label_cic_range.
- Nếu bạn cần phân phối lớp theo mục tiêu huấn luyện cân bằng, dùng label (đã quota).
- Khi đánh giá chất lượng dữ liệu, nên kiểm tra đồng thời cả raw features (Growth_value, CV_value, Spike_ratio) và component scores.

### EN
- If you need strict CIC-range labels, prefer label_cic_range.
- If you need quota-balanced classes for training, use label (already quota-balanced).
- For data-quality checks, validate both raw features (Growth_value, CV_value, Spike_ratio) and component scores.
