Phiên bản: 1.0
Ngày cập nhật: 2026-04-17

============================================================
I. MỤC TIÊU & PHẠM VI
============================================================
Mục tiêu frontend:
1. Banker theo dõi rủi ro tín dụng, dữ liệu giao dịch, hồ sơ vay và khuyến nghị AI để ra quyết định.
2. User đăng nhập, xem hồ sơ doanh nghiệp/cá nhân, tạo yêu cầu vay và theo dõi trạng thái yêu cầu.

Phạm vi use case đầu vào đã phân tích:
1. Banker_usecase:
- managementdashboard.txt
- viewcustomer_transactiondata.txt
- viewloanrequests.txt
- view_user_profile.txt
2. User_Usecase:
- loan_user_api_docs.txt

============================================================
II. KIẾN TRÚC THÔNG TIN (INFORMATION ARCHITECTURE)
============================================================
A. Nhánh Banker
1. Đăng nhập Banker
2. Management Dashboard (Credit Risk Dashboard)
3. Transaction Dashboard (POS Revenue & Cashflow)
4. Loan Requests + AI Recommendation (chung 1 interface)
5. User Profile/Tình trạng doanh nghiệp (drill-down)

B. Nhánh User
1. Đăng nhập User
2. Trang hồ sơ của tôi
3. Tạo yêu cầu vay
4. Danh sách yêu cầu vay của tôi
5. Chi tiết yêu cầu vay

============================================================
III. WIREFRAME CHI TIẾT - BANKER APP
============================================================

1) Màn hình: Banker Login
Mục tiêu:
- Xác thực nhân viên ngân hàng.

Layout wireframe:
- Header logo ngân hàng.
- Card login ở giữa màn hình:
	- Input: user_id
	- Input: mật khẩu
	- Nút: Đăng nhập
	- Link: Quên mật khẩu (tuỳ chọn)
- Footer: chính sách bảo mật.

Trạng thái:
- Loading khi submit.
- Error inline khi sai tài khoản/mật khẩu.

------------------------------------------------------------
2) Màn hình: Management Dashboard (Credit Risk)
Mục tiêu:
- Theo dõi tổng quan danh mục tín dụng và mô phỏng ngưỡng phê duyệt.

Layout wireframe:
- Top bar:
	- Tên hệ thống
	- Bộ chọn khoảng thời gian dữ liệu
	- Nút reset bộ lọc
	- Avatar banker
- Sidebar trái (Global Filters):
	- Industry (dropdown)
	- Business Type (dropdown)
	- Location (dropdown)
	- Regime (dropdown)
	- Years in Business (slider)
	- CIC Score Range (range slider)
- Main content:
	- Row 1: KPI cards
		- Average CIC Score
		- Average Default Probability
		- % High Risk Customers
	- Row 2:
		- Histogram CIC_SCORE
		- Pie chart Risk Label (LOW/MEDIUM/HIGH)
	- Row 3:
		- Bar chart Industry vs Avg CIC
		- Bar chart Industry vs Avg Default Probability
	- Row 4:
		- Scatter Growth_score vs CIC_SCORE (size: Revenue_mean_30d, color: label)
		- Scatter CV_score vs CIC_SCORE
	- Row 5:
		- Bar chart Spike_ratio (bucketed)
		- Pie chart Regime
	- Row 6 (Decision Simulation):
		- Scatter CIC_SCORE vs default_probability
		- Slider CIC cutoff (500-700)
		- Dynamic outputs:
			- Approval Rate
			- Average Risk (approved)
			- Expected Loss
	- Row 7:
		- Table customer summary:
			- customer_id
			- CIC_SCORE
			- default_probability
			- label
			- action: Xem chi tiết
	- Row 8:
		- Customer Detail Panel (khi chọn 1 customer)
			- Radar chart: Growth_score, CV_score, Spike_score, Txn_freq_score
			- Insight text tóm tắt

Tương tác bắt buộc:
1. Cross-filter toàn dashboard khi chọn bất kỳ chart/filter nào.
2. Click HIGH risk trên pie chart phải filter toàn bộ.
3. Lasso trên scatter phải lọc table customer.
4. Click 1 customer ở table mở panel chi tiết.

------------------------------------------------------------
3) Màn hình: Transaction Dashboard (POS Revenue & Cashflow)
Mục tiêu:
- Theo dõi doanh thu, dòng tiền và hành vi giao dịch để hỗ trợ phân tích tín dụng.

Layout wireframe:
- Top bar + breadcrumb: Banker / Transaction Dashboard
- Filter panel:
	- customer_id
	- merchant_id
	- industry
	- business_type
	- location
	- Date range picker
	- Status (Successful/Failed)
- KPI cards:
	- Total Revenue
	- Total Transactions
	- Avg Transaction Value
	- Success Rate
- Khối chart:
	- Line: Revenue Time Series (toggle Daily/Weekly/Monthly)
	- Line: Transaction Volume Trend
	- Bar: Revenue by Industry
	- Bar: Revenue by Business Type
	- Histogram: Revenue Distribution
	- Pie: Transaction Status
	- Bar: Time-of-Day Pattern
	- Heatmap: Day of Week x Hour
	- Bar: Top customer by revenue
	- Bar: Top merchant by revenue
	- Line: Invoice count over time
	- Bar: Avg Invoice Value
	- Pie + Bar: Bank Flow Distribution
- Bảng cuối trang: Top Transactions
	- Date, Amount, Beneficiary Name, Bank, Invoice ID
	- Sort mặc định: Amount giảm dần

Tương tác bắt buộc:
1. Click Industry filter toàn bộ dashboard.
2. Click 1 điểm ngày trên line chart cập nhật toàn bộ.
3. Click 1 customer hiển thị lịch sử giao dịch chi tiết.

------------------------------------------------------------
4) Màn hình: Loan Requests + AI Recommendation (Chung Interface)
Mục tiêu:
- Xem hồ sơ yêu cầu vay và đọc khuyến nghị AI để quyết định APPROVE/REJECT.

Layout wireframe:
- Split layout 2 cột:
	- Cột trái (60%): Loan Request List + Metrics
	- Cột phải (40%): AI Recommendation Panel

Cột trái:
1. Bộ lọc:
	- customer_id, merchant_id
	- industry, business_type, location
	- risk label/regime
	- CIC range, default_probability range
	- loan_type
2. Bảng metrics (theo yêu cầu):
	- customer_id
	- merchant_id
	- name
	- age
	- industry
	- business_type
	- years_in_business
	- location
	- created_at
	- Revenue_mean_30d
	- Revenue_mean_90d
	- Txn_frequency
	- regime
	- Growth_value
	- Growth_score
	- CV_value
	- CV_score
	- Spike_ratio
	- Spike_score
	- Txn_freq_score
	- Years_score
	- Industry_score
	- CIC_SCORE
	- label
	- label_quantile
	- label_cic_range
	- default_probability
	- loan_type
3. Mỗi dòng có action:
	- Xem AI
	- Xem hồ sơ doanh nghiệp
	- Mở Decision modal

Cột phải (AI Recommendation Panel):
1. Header:
	- customer_id
	- trạng thái hồ sơ
2. Tabs:
	- Bank View (bắt buộc dùng report_text_bank)
	- User View (tùy chọn hiển thị report_text_user)
3. Nội dung:
	- Parse markdown string thành các section rõ ràng
	- Hiển thị recommendation cuối cùng: APPROVE/REJECT
	- Hiển thị hạn mức đề xuất (nếu có)

Lưu ý triển khai quan trọng:
1. Theo yêu cầu nguồn dữ liệu: "chỉ parse string cho bank".
2. Mặc định render report_text_bank cho Banker.
3. Nếu có report_text_user thì để tab phụ hoặc ẩn theo policy nội bộ.

Decision Modal wireframe:
- Thông tin nhanh:
	- CIC_SCORE, default_probability, label, loan_type
- Radio:
	- APPROVE
	- REJECT
- Input:
	- approved_limit (nếu APPROVE)
	- reason (bắt buộc)
- Nút:
	- Xác nhận
	- Hủy

Layout wireframe:
- Header thông tin định danh:

IV. WIREFRAME CHI TIẾT - USER APP
============================================================

1) Màn hình: User Login
Theo API:

Layout wireframe:
- Card giữa màn hình:
	- Input user_id
	- Input password
	- Nút Đăng nhập

------------------------------------------------------------
2) Màn hình: Hồ sơ của tôi
Theo API:

Layout wireframe:
- Thẻ profile:
	- customer_id
	- merchant_id
	- name
	- age
	- industry
	- business_type
	- years_in_business
	- location
	- created_at
- Nút chính:
	- Tạo yêu cầu vay mới
	- Xem danh sách yêu cầu vay

------------------------------------------------------------
3) Màn hình: Tạo yêu cầu vay
Theo API:
- GET /loan-purposes
- POST /loan-requests

Layout wireframe:
- Form card:
	- Select Mục đích vay (purpose)
		- Nguồn dữ liệu: loan-purposes.items[].name
	- Input số tiền đề nghị vay (requested_amount)
	- Currency (default VND)
	- Textarea ghi chú (note)
	- Nút Gửi yêu cầu

Validation UX:
1. purpose bắt buộc và chỉ nhận giá trị enum hợp lệ.
2. requested_amount > 0.
3. Hiển thị lỗi 422 ngay dưới field purpose.

------------------------------------------------------------
4) Màn hình: Danh sách yêu cầu vay của tôi
Theo API:
- GET /loan-requests

Layout wireframe:
- Filter ngang:
	- status
	- từ ngày/đến ngày (nếu có)
- Table:
	- request_id
	- purpose
	- requested_amount
	- currency
	- status
	- created_at
	- action: Xem chi tiết
- Pagination: page, page_size

------------------------------------------------------------
5) Màn hình: Chi tiết yêu cầu vay
Theo API:
- GET /loan-requests/{request_id}

Layout wireframe:
- Summary card:
	- request_id, status, created_at, updated_at
- Detail card:
	- purpose
	- requested_amount
	- currency
	- decision_note
- Timeline trạng thái:
	- SUBMITTED -> IN_REVIEW -> APPROVED/REJECTED

V. DESIGN SYSTEM CHUNG CHO FRONTEND
============================================================
1. Grid:
- Desktop: 12 cột
- Tablet: 8 cột
- Mobile: 4 cột

2. Breakpoints:
- >=1200: desktop
- 768-1199: tablet
- <768: mobile

3. Quy chuẩn component:
- Bộ lọc dùng dạng chips + dropdown + range slider.
- Bảng dữ liệu sticky header, hỗ trợ sort/filter/search.

	- SUBMITTED: neutral
	- IN_REVIEW: warning
	- APPROVED: success
	- REJECTED: danger

4. Empty/Error/Loading states:
- Empty: minh họa + CTA quay lại bộ lọc.
- Error: mã lỗi + thông điệp + nút thử lại.
- Loading: skeleton cho cards/charts/table.

X.1) Màn hình: Transaction Raw Table
Nguồn: random_transaction_10_firms.csv

A. Cột hiển thị chính (mặc định nhìn thấy)
1. customer_id
2. merchant_id
3. name
4. industry
5. business_type
6. location
7. Status
8. Amount
9. Date
10. Time
11. Beneficiary Bank
12. Transaction Code
13. ID hóa đơn
14. Tổng giá

B. Cột phụ (ẩn mặc định, bật trong column settings)
1. age
2. years_in_business
3. created_at
4. Beneficiary Account
5. Beneficiary Name
6. Transfer Content
7. Reference Code
8. Ngày

C. Format hiển thị chuẩn
1. Amount, Tổng giá: format số có dấu phân tách hàng nghìn, đơn vị VND.
2. Date, Ngày: normalize về dd/MM/yyyy.
3. Time: hiển thị HH:mm.
4. Status: badge màu
	 - Successful transaction: xanh
	 - Failed transaction: đỏ
5. Beneficiary Account: mask bớt số, chỉ hiện 4 số cuối.

D. Filter chuẩn
1. customer_id (text)
2. merchant_id (text)
3. industry (dropdown)
4. business_type (dropdown)
5. location (dropdown)
6. Status (multi-select)
7. Date range
8. Amount range

E. Sort chuẩn
1. Date giảm dần (mặc định)
2. Amount tăng/giảm
3. Tổng giá tăng/giảm

F. Row action
1. Xem chi tiết giao dịch (open drawer)
2. Xem lịch sử theo customer_id (jump sang filtered view)

G. Detail drawer structure
1. Header: Transaction Code, Status, Date-Time
2. Khối người nhận: Beneficiary Name, Beneficiary Account, Beneficiary Bank
3. Khối nội dung: Transfer Content, Reference Code, Invoice ID
4. Khối giá trị: Amount, Tổng giá

          content:
            application/json:
              schema:
